# 03 - Scenario Playbook

## How to use this playbook

Select one primary customer scenario, define a 90-day target state, and execute with clear stage gates.

## Scenario A: Identity and endpoint risk reduction

Typical triggers:

- Increased phishing/malware incidents
- Remote/hybrid workforce with inconsistent endpoint controls
- Limited internal security operations capacity

Execution stages:

1. Discovery and baseline validation
2. Priority control design
3. Pilot rollout
4. Full deployment wave
5. Stabilization and KPI tracking

Success metrics:

- MFA and identity policy coverage
- Endpoint protection policy compliance
- Incident response time improvement

## Scenario B: Data protection and compliance readiness

Typical triggers:

- Customer handles sensitive financial/legal data
- Need to improve data handling governance
- Audit readiness gaps

Execution stages:

1. Data and process discovery
2. Policy and classification design
3. Controlled pilot and process adoption
4. Compliance operating rhythm setup

Success metrics:

- Data labeling adoption rate
- Sensitive data exposure reduction
- Audit evidence readiness quality

## Scenario C: Secure growth for resource-constrained SMB

Typical triggers:

- Fast headcount growth
- New location or acquisition activity
- IT team stretched across priorities

Execution stages:

1. Risk-prioritized architecture plan
2. Minimum viable controls rollout
3. Managed operations handoff

Success metrics:

- Security control coverage growth
- Service desk ticket reduction for security-related issues
- Leadership confidence score in security posture

## Stage-gate quality checks

- Gate 1: Business problem and success criteria signed off
- Gate 2: Target architecture and scope approved
- Gate 3: Pilot outcomes validated
- Gate 4: Adoption indicators above minimum threshold
